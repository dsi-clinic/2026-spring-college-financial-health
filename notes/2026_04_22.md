## Key Work Done

### Shared utility module (`scripts/utils.py`)

Created `scripts/utils.py` with `normalize_opeid()`. The function was previously
duplicated across three files:
- `build_panel.py` (function definition)
- `identify_closed_colleges.py` (identical copy)
- `replicate_table5.py::merge_frc` (inlined)

All three now import from `utils.py`.

### `scripts/build_panel.py` — 6 fixes

1. **CPI deflator loop**: `panel["year"].map(cpi_deflator)` was recomputed once per
   dollar column (11 times total). Moved outside the loop — computed once, reused.

2. **Dead `sc()` helper**: Replaced 11 repeated
   `panel.get(col, pd.Series(np.nan, index=panel.index))` calls with a one-line local
   helper `sc = lambda col: panel.get(col, nan)`.

3. **Dead variable `enr`**: `enr = panel.set_index(...)["enroll"]` in
   `add_enrollment_changes` was assigned but never read (the `enroll_prev` computation
   used a separate call). Removed.

4. **Dead variables `neg_margin`, `decline`**: Both computed a boolean series that was
   immediately recomputed inside the following lambda. Removed both assignments.

5. **Unused loop variable `uid_col`**: Loop in `add_rolling_indicators` destructured
   a 3-tuple but `uid_col` was never used (the groupby hardcodes `"unitid"`). Renamed
   to `_`.

6. **Boolean comparison**: `panel[panel["is_closed"] == True]` → `panel[panel["is_closed"]]`.

### `scripts/identify_closed_colleges.py` — 3 fixes

1. **Column-pruned parquet read**: `pd.read_parquet(PANEL_PATH)` loaded all 76 columns
   of the 150k-row panel. Now reads only the 7 columns actually used.

2. **Single sort**: `load_panel_summary` called `sort_values("year")` twice (once before
   `.groupby().last()`, once before `.groupby().first()`). Consolidated to one sort.

3. **Removed duplicate `normalize_opeid`**: Now imported from `utils.py`.

### `scripts/replicate_table5.py` — 2 fixes

1. **O(n²) list search**: `paper_rows.index(row)` inside `for row in paper_rows` is
   O(n) per iteration. Replaced with `enumerate` — `for i, row in enumerate(paper_rows)`.

2. **Inline normalize_opeid**: Replaced 4-line inline normalization in `merge_frc` with
   `normalize_opeid(panel["opeid"])` from `utils.py`.

### `scripts/predict_new_data.py` — 1 fix + 1 doc

1. **Unnecessary `.copy()`**: `_get_feature_matrix` called `_make_sector_dummies(df.copy())`,
   doubling memory for large inputs. XGBoost creates its own `DMatrix` copy anyway.
   Removed the `.copy()`.

2. **Documented year-dummy behavior**: Added a note explaining that year dummy columns
   (`yr_*`) will be NaN for out-of-sample institutions, and that this is handled correctly
   by XGBoost's native missing-value path.

## College Scorecard Integration

### What was discovered

Investigated `CURROPER` and `HCM2` availability in the Scorecard bulk files — both are
fully suppressed (`NA`) across all years 1996–2024. They are only exposed in the
current-snapshot API, not historically. No year-by-year HCM2 signal is available without
a direct FSA data request.

What IS available and well-populated for sectors 6/9:
- `UGDS` — undergrad headcount enrollment (~91% coverage)
- `TUITFTE` — tuition revenue per FTE (~95% coverage)
- `INEXPFTE` — instructional expenditure per FTE (~95% coverage)
- `PCTPELL` — share of students receiving Pell grants (~65% coverage)

### New file: `scripts/merge_scorecard.py`

Reads directly from the zip without extracting (memory-efficient). Maps IPEDS year `Y`
to Scorecard filename `MERGED{Y}_{Y+1 mod 100:02d}_PP.csv`. Extracts the four columns
above for 2002–2022 and saves to `data/raw/scorecard/scorecard_panel.parquet`
(149k rows, 10,879 unique institutions).

`make merge-scorecard` target added to Makefile.

### `build_panel.py` updates

- Added `load_scorecard()` function; loads panel once before the year loop
- Per-year merge fills IPEDS `enroll` gaps with Scorecard `UGDS`
- `sc_tuitfte` and `sc_inexpfte` added to CPI deflation list
- `sc_tuitfte_real`, `sc_inexpfte_real`, `sc_pctpell` (+ L2/L3 lags) added to lag targets
- Enrollment coverage improved: **90% → 95.7%**

### `replicate_table5.py` updates

Added 7 Scorecard features to `EXTENDED_VARS` (used in the "All Controls" XGBoost model F):
`sc_tuitfte_real`, `sc_tuitfte_real_l2`, `sc_tuitfte_real_l3`,
`sc_inexpfte_real`, `sc_inexpfte_real_l2`, `sc_pctpell`, `sc_pctpell_l2`

Added zero-denominator guards throughout `add_derived_variables` in `build_panel.py`:
- `debt_to_assets`: `lt_debt / assets.replace(0, np.nan)`, clipped to [0, 20]
- `unrestricted_na_ratio`: `unrestr / assets.replace(0, np.nan)`

- `dcoh`: `cash_sti / (exp.replace(0, np.nan) / 365)`
- All expense/revenue shares guarded similarly

### 4. Final deliverable: IPEDS-only prediction function — DONE

Created `scripts/college_risk.py` with two callable functions:

**`predict(panel, year=None)`** — primary interface
```python
from college_risk import predict
results = predict("analysis/panel/institution_year_panel.parquet", year=2022)
# Returns DataFrame sorted by closure_prob_1yr with columns:
# unitid, instnm, stabbr, sector, year, closure_prob_1yr, closure_prob_3yr
```

**`top_at_risk(panel, year=None, n=50)`** — convenience wrapper
```python
from college_risk import top_at_risk
top = top_at_risk(n=20)  # uses most recent panel year automatically
```

Also callable as a CLI:
```
uv run python scripts/college_risk.py --year 2022 --top 20
uv run python scripts/college_risk.py --year 2022 --output predictions.csv
```

Key implementation notes:
- Filters to private sectors (4–9) only — model was trained on those exclusively
- Creates sector, year, and state dummies from the panel's categorical columns
- Aligns features to the model's stored `feature_names`, adding any missing as NaN
- XGBoost handles NaN natively — no imputation required
- Deduplicates feature lists (`dict.fromkeys`) — fixes a bug where `enroll_l2`/`enroll_l3`
  appeared in both `SELECT_CONTINUOUS` and `EXTENDED_VARS`, causing XGBoost to choke
  on duplicate column names. Also fixed in `replicate_table5.py`.
