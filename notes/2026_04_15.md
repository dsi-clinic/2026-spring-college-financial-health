## Key Work Done

### IPEDS Finance URL Fix

The NCES download URL format for older finance files (2002-2020) uses a two-digit year-range
format: `F{yr-1 last 2d}{yr last 2d}` (e.g., `F0102_F1A` for FY2002). The formula is:
- `yr2d = (year - 1) % 100` zero-padded to 2 digits
- `next2d = year % 100` zero-padded to 2 digits

Initial bug had `yr2d = year % 100` which put each file one year ahead. All 22 years
of finance data (F1A, F2, F3) were re-downloaded with the correct formula.

### Panel Construction (`scripts/build_panel.py`)

Built from scratch — 150,207 institution-year rows, 76 feature columns, 2002-2022.

Key findings:
- **F1A file = Public (GASB)**, F2 file = Private NP (FASB), F3 = For-profit (FASB corporate)
  — confirmed via Harvard cross-check (f2a01 = $62B total assets ✓)
- **Enrollment old format** (2002-2009): uses FTPT=3, FUNCTCD=10, FSTAT=0 → EAPTOT
  Not obvious from documentation; found via column inspection
- **F3 total expenses**: no direct column; derived as `total_rev - (net_assets_end - net_assets_begin)`
- **L2/L3 lags**: institutions that will close often DON'T have current-year finance data,
  but DO have L2/L3 data — so lags are the right features for OLS "non-missing" models

### Table 5 Replication

AUC 0.73-0.77 vs paper's 0.87-0.90. Gap is explained:
- 80% of closures are in sectors 6, 9 (less-than-2-yr vocational) with NO finance data
- Paper supplemented with College Scorecard (which we can't access without API key)
- Qualitative pattern correct: XGBoost > OLS, All Controls > Select Controls

**Critical insight**: Institutions that close in year T typically don't file year-T IPEDS
surveys. Using LEAD outcomes (predict year T+1 closure from year T features) gives
meaningful coverage. Even with leads, OLS models fail for sectors 6/9 because there's
zero finance data in the "non-missing" sample.
